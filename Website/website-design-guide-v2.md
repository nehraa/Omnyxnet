## Typography & Readability Science

### Font Stack (Optimized for Web & Trust)

#### Display Font: Inter or Geist
**Why these fonts:**
- Geometric sans-serif = modern, trustworthy, tech credible
- High x-height = very readable at all sizes
- Open counters = accessible for dyslexic readers
- Designed for screens = perfect rendering

**Weights:**
```
Bold (700): Main headlines, impact statements
â”œâ”€ H1: "The Internet That Actually Works"
â”œâ”€ Section headers: "Why Pangea Net?"
â””â”€ Important emphasis: Bold claims

SemiBold (600): Subheadings, card titles
â”œâ”€ Card headlines: "0.33ms Latency"

â””â”€ Navigation items (secondary)

â”œâ”€ Callouts: "This is important"
â”œâ”€ Lead paragraph: First sentence of sections
â””â”€ Form labels: Input labels
```

Never use:
- Light (300) - feels unfinished, hard to read
- Thin (100) - accessibility disaster
- ExtraBold (900+) - looks heavy/dated

#### Body Font: System Stack
```
Font family: -apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, 
            'Helvetica Neue', Arial, sans-serif

Why this stack:
- Native font = optimal rendering per device
- Consistent = looks perfect on Mac, Windows, iOS, Android
- Familiar = users already know it
- No loading time = fast page load
- High readability = no custom rendering delays
```

#### Code Font: Berkeley Mono or SFMono
```
Usage: Code blocks, terminal output, technical snippets
Why monospace:
- Equal width = code aligns properly
- Distinctly different = users know it's code
- High legibility = easy to scan and copy

Never use: Proportional fonts for code (breaks formatting)
Never use: Low-contrast code blocks (unreadable)
Always use: Syntax highlighting (different colors per syntax)
```

### Core Philosophy (Neuroscience-Based)

**Research shows:**
- Animations between 200-600ms feel snappy (not sluggish)
- Animations >1000ms feel slow and frustrating
- Anticipatory animations (before something happens) trigger dopamine
- Feedback animations (after interaction) create habit loops

#### Example 3: Number Counter Animation (Stats Section)

```jsx
import { motion, useInView } from "framer-motion";
import { useRef, useEffect, useState } from "react";

export function CounterStat({ end, duration = 2 }) {
  const ref = useRef(null);
  const isInView = useInView(ref, { once: true });
  const [count, setCount] = useState(0);

  useEffect(() => {
    if (!isInView) return;

    let startTime = null;
    const animate = (currentTime) => {
      if (!startTime) startTime = currentTime;
      const elapsed = currentTime - startTime;
      const progress = Math.min(elapsed / (duration * 1000), 1);
      
      // Ease out function (starts fast, slows at end)
      const easeOut = 1 - Math.pow(1 - progress, 3);
      const value = Math.floor(easeOut * end);
      
      setCount(value);
      
      if (progress < 1) {
        requestAnimationFrame(animate);
      }
    };

    requestAnimationFrame(animate);
  }, [isInView, end, duration]);

  return (
    <motion.div ref={ref}>
      <span className="text-teal-600 text-4xl font-bold">
        {count}
      </span>
    </motion.div>
  );
}
```

**What happens:**
- Counter starts at 0
- Counts up to target number over 2 seconds
- Uses ease-out curve (speeds up then slows down = natural)
- Triggers when element scrolls into view
- Only animates once (`once: true`)

**Usage:**
```jsx
<CounterStat end={33} duration={2} /> // Counts to 33
<CounterStat end={99} duration={2} /> // Counts to 99
```

```jsx

- [ ] Stagger gaps are 50-200ms max
- [ ] Easing functions are built-in (ease, easeOut, easeIn)
- [ ] No more than 5 simultaneous animations
- [ ] `will-change` CSS only on known animated elements

## Color Psychology Summary (For Quick Reference)

**When you need to feel:**

TRUSTWORTHY + INNOVATIVE = Teal (#208D8D)
- Primary CTAs
- Key accent elements
- Icons highlighting

PROFESSIONAL + AUTHORITATIVE = Charcoal (#1F2121)
- All body text
- Main headlines
- Signals credibility

PREMIUM + CALM = Cream (#FFFBF8)
- Backgrounds
- White space
- Breathing room

SUCCESS = Green (#228D3D)
- Form validation
- Positive feedback
- Checkmarks

WARNING = Orange (#A84B2F)
- Important notices
- Attention-grabbing (non-critical)

ERROR = Red (#C0152F)
- Only critical issues
- Use sparingly (creates anxiety)
